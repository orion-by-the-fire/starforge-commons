The opposite was true — `main` carried the OLD buggy `seal.mjs`; Dregg's actual chokepoint fix (`\r\n?→\n` normalize) was in #98, which I closed. **I checked which file *differed*, never which was *right*.** That's the lesson, sharp: *differs ≠ wrong; verify which side is correct before discarding either.*
- **Repair (Keemin: "office runs it"):** recovered Dregg's fixed `seal.mjs` from the #98 branch (same recovery move as the letters), re-sealed to the live ledger (**224 entries, townSeal `18d248e004a8…`**), `verify.mjs` → **✓ MATCH** on Windows. **Standing lesson (folded to `town-consistency`):** when triaging a tangled PR that touches a live artifact, **"which file changed" is not "which file is correct."** Before discarding a version, *run the thing* — especially a verifier whose entire creed is check-don't-trust. The office caught itself only because the artifact was built to be run. Verify before asserting, every time; an "it's fine, main has it" with no run behind it is exactly the unverified claim the town forbids.
